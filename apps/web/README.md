# Webview

This app is a Vite SPA using TanStack Router (file-based routes), Tailwind CSS v4 (via Vite plugin), and shadcn/ui.

## Commands

- `bun install` - install dependencies
- `bun run dev` - start the local dev server
- `bun run generate:backend` (from repo root) - refresh backend Convex `_generated` API before web typecheck/build if backend functions changed
- `bun run lint` - run ESLint
- `bun run preview` - preview the production bundle

## Routing

- Routes live in `src/routes`
- `src/routes/__root.tsx` defines the app shell/root layout
- `src/routes/_auth/sign-in.tsx` and `src/routes/_auth/callback.tsx` handle WorkOS auth redirects
- `src/routes/index.tsx` defines the `/` route
- `src/routes/components.$componentId.tsx` defines the component metadata route
- `src/routeTree.gen.ts` is generated by TanStack Router (do not edit)

## Backend Integration

- Set required env vars in `apps/web/.env.local`:
  - `VITE_CONVEX_URL` (typically the same as `CONVEX_URL` in `apps/backend/.env.local`)
  - `VITE_WORKOS_CLIENT_ID` (WorkOS client/application ID)
  - `VITE_WORKOS_REDIRECT_URI` (for example `http://localhost:5173/callback`)
- Web imports backend generated API types directly from `apps/backend/convex/_generated/api`

## shadcn/ui

- Config is in `components.json`
- Add new components with `bunx shadcn@latest add <component>`
- Shared utilities are in `src/lib/utils.ts`
